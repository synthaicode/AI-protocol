# ログ

## 概要

本ドキュメントは、AI協働プロトコルに関する**決定履歴**と**保留事項**を記録する。

### 原則（Building Shared Memory with AI より）

- **AIがログを書く** - 人間は承認のみ
- **事実のみ記録** - 解釈を加えない
- **イベントベースで構造化** - 時系列で追跡可能
- **条件ベースの保留管理** - 日付ではなく条件で再提示

### 保留事項の運用

- **条件ベース**: 日付ではなく、具体的な条件で管理
- **AIの責任**: 条件充足を監視し、自動的に再提示
- **ユーザーの責任**: 再提示された項目を判断

**例**:
```markdown

|----|------|---------------|--------|------|
| P001 | デプロイ先の選定 | インフラ担当者との MTG 完了 | 2026-01-15 | 要件未確定 |
```

### 決定履歴の記録形式

各決定は以下の形式で記録：

- **日時**: YYYY-MM-DD HH:MM（時刻は重要な場合のみ）
- **Event**: 何が起きたか（イベント名）
- **Decision**: 何を決定したか（決定内容）
- **Stated Reason**: なぜそう決定したか（理由）
- **出所**: 情報の出所タグ

- 保留事項が解消されたら、この表に移動
- 決定内容を「決定履歴」にも記録
- 保留事項の表からは削除

```markdown
|----|------|--------|------|----------|
| P001 | デプロイ先の選定 | 2026-01-20 | 決定 | Vercelを採用 |
```

## ログの更新ルール

### AIの責任

1. **自動記録**: 決定が発生したら即座に記録
2. **条件監視**: 保留事項の pending_until を常時監視
3. **再提示**: 条件充足時に能動的に提案
4. **事実記録**: 解釈を加えず、事実のみ記録

### ユーザーの責任

1. **承認**: AIが記録した内容を確認・承認
2. **訂正**: 誤りがあれば訂正指示
3. **判断**: 再提示された保留事項を判断
