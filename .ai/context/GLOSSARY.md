# 用語集

## 概要

本ドキュメントは、AI協働プロトコルで使用される**正式用語**と**その定義**を管理する。

AIは、聞き返しや出力の際に必ずこの用語集の正式用語を使用する。

## 用語の構造

各用語は以下の情報を持つ：

- **正式用語**: このプロトコルで使用する正式な用語
- **別名**: ユーザーが使う可能性のある別の表現
- **定義**: 用語の明確な定義
- **出所**: この用語の情報源 ([明示][推論][仮定][補完])

## 用語の追加・変更プロセス

### 新規用語の追加

1. AIが会話中に新しい用語を発見
2. 用語集への追加を提案
   ```
   【用語追加提案】
   正式用語: XXX
   別名: YYY, ZZZ
   定義: ...
   出所: [明示]
   ```
3. ユーザーが承認/訂正
4. GLOSSARY.mdに追加
5. statusを draft → approved に変更

### 用語の変更

1. AIまたはユーザーが不整合を発見
2. 変更提案
   ```
   【用語変更提案】
   現在: XXX（定義: ...）
   変更後: YYY（定義: ...）
   理由: ...
   ```
3. ユーザーが承認
4. GLOSSARY.mdを更新
5. LOG.mdに変更履歴を記録

### 用語の統一

AIは以下の責任を持つ：

- **検出**: 会話から新しい用語を発見
- **提案**: 「この用語を追加しますか？」と確認
- **統一**: ドキュメント内の用語ブレを指摘
- **変換禁止**: 勝手な言い換えをしない

## AIの使用ルール

### 聞き返し時

❌ **悪い例**:
```
ユーザー: 「バウンダリを設定して」
AI: 「バウンダリを設定します」
```

✅ **良い例**:
```
ユーザー: 「バウンダリを設定して」
AI: 「ガードレール（境界線）を設定する、という理解で正しいですか？」
```

### 出力時

❌ **悪い例**:
```
AI: 「制約条件を確認しました」
```

✅ **良い例**:
```
AI: 「ガードレール（境界線）を確認しました」
```

### 新規用語の扱い

用語集にない用語が出現した場合：

1. その会話では仮に使用
2. セッション終了時に用語追加を提案
3. ユーザー承認後、GLOSSARY.mdに追加

## まとめ

- AIは**必ず正式用語**で聞き返し・出力する
- 新出用語は**積極的に提案**する
- 用語のブレは**指摘し統一**する
- 勝手な言い換えは**禁止**

これにより、人間とAIの間で言葉が揃い、認識が揃い、行動が揃う。
